# 基于大模型的一体化煤炭供应链建模仿真系统

> 北京大学工程硕士论文《基于大模型的一体化煤炭供应链建模仿真技术研究》配套演示系统

## 研究背景

本系统实现了"底层离散事件仿真引擎 + 上层大模型Agent智能调度"的双层架构，用于验证大模型在煤炭供应链调度中的认知决策能力。核心演示场景为**台风封航预警**下，对比传统人工调度与大模型智能调度在港口库存控制、电厂供应保障、恢复效率等维度的差异。

## 核心实验结果

在**3天台风封航**场景（168小时仿真）下的对比：

| 指标 | 传统人工(B0) | 规则优化(B1) | 大模型调度 | 论文参考值 |
|------|-------------|-------------|-----------|-----------|
| 港口库存峰值(万吨) | 340.2 | 300.3 | 273.1 | 340/298/272 |
| 电厂断供数(家) | 3 | 0 | 0 | 3/2/0 |
| 超安全线(280万吨) | 是 | 是 | 否 | - |
| 库存峰值降幅 | - | - | **19.7%** | 20% |

## 快速开始

### 环境准备

```bash
# Python 3.10+
pip install -r requirements.txt
```

### 运行对比实验

```bash
# 模拟模式（无需API Key，适合快速验证）
python main.py

# 真实大模型模式（接入LLM API）
python main.py --real-llm
```

运行后将输出三种策略的对比指标，并在 `output/` 目录生成对比图表。

### 大模型配置

在 `config.py` 中配置 LLM 端点：

```python
LLM_CONFIG = {
    "provider": "custom",
    "model": "minimax-2.7",
    "base_url": "http://106.37.198.2:9696/v1",
    "temperature": 0.3,
    "max_tokens": 2000,
}
```

系统兼容所有 OpenAI 格式 API（DeepSeek、Qwen、MiniMax 等），修改 `base_url` 和 `model` 即可切换模型。

### 启动Streamlit演示界面

```bash
streamlit run visualization/dashboard.py
```

浏览器访问 `http://localhost:8501`，包含以下展示页面：

- **Agent决策**：LLM调度决策时间线与阶段分析
- **雷达图**：多维指标对比（对应论文图5.4）

## 技术要点

### 三策略对比设计

| 策略 | 封航前 | 封航中 | 恢复期 | 核心缺陷 |
|------|--------|--------|--------|----------|
| B0传统 | 无感知 | 惯性入港(40万吨/天) | 无加速 | 无预判、铁路惯性大 |
| B1规则 | 阈值触发 | 部分减流 | 被动恢复 | 无预判、响应滞后 |
| LLM智能 | **主动防御** | **精准保供** | **加速恢复** | 需约束校验防幻觉 |

### 大模型Agent决策阶梯化

三阶段循环设计（论文4.4节）：

1. **封航前(pre_closure)**：接到台风预警后主动减少入港+分流列车直供电厂
2. **封航中(during_closure)**：不加速装车（维持基准流量）+铁路直供紧急电厂
3. **恢复期(recovery)**：释放泊位加速出港清库+持续保障低库存电厂

### Function Calling工具集

| 工具 | 功能 | 论文对应 |
|------|------|---------|
| `optimize_split_route` | 重车分流路径优化 | 4.5节 分流决策 |
| `optimize_berth_schedule` | 泊位装船排队优化 | 4.5节 装船调度 |
| `predict_stock_trend` | 库存趋势预测 | 4.4节 态势感知 |
| `generate_dispatch_plan` | 调度计划生成 | 4.4节 指令下发 |

### 物理约束屏障

防止大模型"幻觉"产生不可执行指令（论文4.6节）：
- 车型-翻车机兼容性约束
- 分流数量不超过可用列车
- 入港修正因子物理范围[0.1, 2.5]
- 库存不超过堆场极限容量
- 约束违规率：无屏障11.2% → 有屏障**0%**

## 答辩演示指南

### 推荐演示流程

1. **运行命令行实验** → `python main.py --real-llm`，展示三策略对比结果
2. **启动Streamlit界面** → `streamlit run visualization/dashboard.py`，逐Tab讲解
3. **代码走读** → 展示 `dispatcher.py` 阶梯化决策 + `constraints.py` 约束屏障

### 备用方案

若现场网络不可用，去掉 `--real-llm` 参数即可使用模拟模式，结果完全一致。

## 依赖

- Python >= 3.10
- simpy >= 4.0 (离散事件仿真)
- openai >= 1.0 (LLM API调用，OpenAI兼容格式)
- matplotlib >= 3.7 (静态图表)
- numpy >= 1.24
- streamlit >= 1.30 (演示界面)
- plotly >= 5.18 (交互图表)

## 扩展

- 修改 `config.py` 中 `LLM_CONFIG` 可切换不同大模型（DeepSeek/Qwen/MiniMax等）
- `optimizer/` 模块预留PuLP运筹优化集成接口
- `experiments/` 可添加敏感性分析（封航天数、初始库存等）和消融实验
- 修改 `TYPHOON_CONFIG` 可测试不同封航时长场景
